# 贡献指南 · Contributing

感谢你愿意为 **Legal-Skills-Chinese** 贡献技能。本指南说明如何提交一个新技能、我们如何评审,以及一份合格 `SKILL.md` 应当具备的要素。

---

## 一、提交方式

我们提供两条路径,任选其一。

### 路径 A · 填表提交(推荐,无需 Git)

点击下方链接,填写表单即可。维护者会据此创建技能目录并并入。

👉 **[提交一个新技能 · Submit a new Skill](../../issues/new?template=submit-skill.yml)**

### 路径 B · Pull Request(熟悉 Git 者)

1. Fork 本仓库。
2. 在仓库根目录新建一个以技能 **slug** 命名的目录(全小写、连字符分隔,例如 `contract-clause-extraction`)。
3. 在该目录内创建 `SKILL.md`。
4. 提交 PR,并在描述中简述该技能解决什么问题、面向什么场景。

---

## 二、`SKILL.md` 规范

每个 `SKILL.md` 必须以 YAML frontmatter 开头:

```yaml
---
name: skill-slug          # 必须与目录名完全一致,全小写连字符
description: |            # 触发条件、适用场景、能力边界
  当用户需要 …… 时触发本技能。
  典型触发场景包括:……
  本技能不处理:……(能力边界)
---
```

正文部分建议包含(顺序可调整):

- **触发条件** —— 什么样的请求应当调用本技能,最好给出关键词与典型句式。
- **能力边界** —— 明确说明本技能**不做**什么,避免越界。
- **操作步骤** —— 分步骤的工作流程。
- **输入 / 输出格式** —— 期望的输入形态与产出结构。
- **法律声明** —— 重申输出为供执业人员审阅的草稿,不构成法律意见。

### 复合能力的额外要求

如果你的技能是**编排型复合能力**(自身不引入新原子能力,而是调度其他技能),请在 `description` 或正文中**显式列出**:

- 它调度了哪些原子能力;
- 调用顺序;
- 各步骤之间如何传递中间结果。

可参考现有的 `legal-judgment-prediction` 与 `judgment-document-generation`。

## 三、法域与质量底线

本库默认面向**中国大陆成文法体系**,贡献时请注意:

- 在成文法体系下,案例原则上不具有普遍约束力(最高人民法院指导性案例除外);**类比推理在刑法定罪量刑、税法课税要件等领域受严格限制甚至禁止**。涉及此类领域的技能必须显式标注该限制。
- 引用法条时应说明其现行有效性与效力层级;不得编造法条或裁判规则。
- 若技能面向其他法域(港澳台、普通法系等),请在 `description` 中明确标注,以免被误用于大陆法域。

---

## 四、我们如何评审

每一份提交都会经过人工评审,依次检查:

1. **安全** —— 不含恶意代码、风险内容或不当依赖。
2. **法律质量** —— 由具备执业背景的评审者核查触发条件是否清晰、能力边界是否准确、是否贴合中国大陆成文法的推理逻辑与实务规范。
3. **实用价值** —— 该技能是否真正为使用者带来可落地的帮助,而非泛化提示词的复述。

通过评审后,我们会将技能并入主分支,并在 README 的相应类别中登记。

## 五、署名与许可

- 你保留对所贡献技能内容的著作权。
- 提交即表示你同意你的贡献以与本仓库一致的方式被收录与展示。
- 本仓库的列表与文档整体以 **CC BY-NC-ND 4.0** 授权;请注意各技能可另行附带其自身的许可条款。

再次感谢你的贡献——正是执业法律专业人员的逐一手写与验证,构成了这套库的专业性来源。
