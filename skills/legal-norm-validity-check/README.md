# 接入法条效力核验 · 北大法宝 MCP

`legal-norm-validity-check`(法律规范效力检查)技能本身**不绑定任何特定数据库**。它定义的是"如何判断一条法条是否现行有效、层级是否正确、与上位法/同位法是否冲突"的方法论。要让核验结果**可溯源、可信**(而非凭模型记忆判断某法条是否被修订/废止),需为运行环境接入一个能**溯源法条、校验现行有效性**的 MCP 服务。

> ⚠️ 本技能风险等级标注为**极高**——引用失效或冲突法条会直接导致法律结论错误。不接入数据库时,任何无法在线核实的效力状态都必须按 SKILL.md §法律声明的要求**如实标注不确定性**,不得伪装成确定结论。

目前已验证可用的是 **[北大法宝 MCP](https://mcp.pkulaw.com)**(500 万+ 法律法规库,实时更新)。其中与本技能最匹配的是**法条/案号识别与溯源**、**修正法条生成幻觉**两类服务。

---

## 一、配置

标准远程 MCP Server,以 Claude Code / Claude Desktop 为例:

```json
{
  "mcpServers": {
    "pkulaw-norm-trace": {
      "url": "https://apim-gw.pkulaw.com/{SERVICE_ID}/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_TOKEN"
      }
    }
  }
}
```

- Token 与 `SERVICE_ID` 在 [mcp.pkulaw.com 控制台](https://mcp.pkulaw.com/console/apps) 获取,**切勿写进 SKILL.md 或提交到仓库**。
- `SERVICE_ID` 在 [MCP 应用中心](https://mcp.pkulaw.com/apis) 选用**法条识别与溯源**或**法律法规检索**类服务(用于拉取法条原文、文号、效力级别、生效/失效日期、修订沿革)。

## 二、技能如何调用接口(接口契约)

`legal-norm-validity-check` 接收"一条或多条已检索到的法条(含出处)",逐条核验后输出效力状态报告。它对接口的期望是——**环境中存在一个能"输入法条标识、返回该法条的现行状态与沿革"的工具**:

| 技能侧(本技能输出/期望) | 接口侧(北大法宝 MCP 提供) |
|---|---|
| 输入:待核验的法条标识(法律名称 + 条号,或引用文本) | **法条/案号识别与溯源**工具 |
| 期望返回:法条原文、文号、**效力级别**、**时效性(现行有效/已修改/已废止)**、修订/被修订关系 | 法规库结构化返回 + 法规变迁/溯源信息 |
| 核验逻辑:据此判断「现行有效性 → 层级正确性 → 与上位法/同位法冲突」(SKILL.md 三步) | —— |
| 反向校验:把模型引用的法条交给「**修正法条生成幻觉**」工具比对,识别条号错位、版本过期、虚构条文 | 修正法条生成幻觉工具 |

也就是说,本技能只依赖两个抽象动作:**"溯源一条法条的现行状态"** 与 **"校验一段法条引用是否真实/现行"**。任何提供等价能力的法规库都可替换北大法宝,无需改动 `SKILL.md`。

> 说明:上表中"法条溯源""修正法条生成幻觉"等为北大法宝公开宣传的**服务类别**;各服务在 MCP 协议中暴露的**确切工具名(tool name)与参数**,请以你连接后实际拉到的工具列表(`tools/list`)或[官方文档中心](https://mcp.pkulaw.com/docs)为准。

## 三、用法示例

```
推理过程中引用了某法条 → 触发 legal-norm-validity-check 技能
→ 技能将法条标识交给 pkulaw-norm-trace MCP 溯源
→ 拿到现行状态(有效/已修订/已废止)、效力级别、修订沿革
→ 技能按「现行有效性 → 层级 → 冲突」三步核验,输出效力状态报告(附理由与置信度)
```

## 四、注意事项

- **时间节点**:法条效力随立法活动随时变化,技能须按 SKILL.md 要求**明确告知核验的时间节点**。
- **官方兜底**:对关键法条,建议同时核对官方[国家法律法规数据库](https://flk.npc.gov.cn)。
- **配套技能**:本库的 `case-retrieval`(案例检索)、`legal-article-retrieval`(法条检索)也可接入北大法宝,配置方式一致,只是换用对应的 `SERVICE_ID`——详见 [`../case-retrieval/README.md`](../case-retrieval/README.md)。
- **最终责任**:无论数据来自哪个库,核验结果均为供执业法律专业人员审阅的草稿,最终判断与责任由执业人员承担。
